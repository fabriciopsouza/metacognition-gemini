# Contribuindo para o Metacognition Framework

Obrigado pelo interesse em contribuir! Este projeto é open-source (CC BY 4.0) e toda ajuda é bem-vinda.

---

## Como Contribuir

### 1. Reportar Bugs ou Problemas

Abra uma [Issue](https://github.com/fabriciopsouza/metacognition-framework/issues) com:

- **Título claro:** ex: "Framework não detecta contexto casual"
- **Descrição detalhada:**
  - O que você fez (pergunta/comando)
  - O que esperava
  - O que aconteceu
- **IA utilizada:** ChatGPT 4, Claude Sonnet, etc
- **Versão do framework:** v2.0, v1.0, etc

### 2. Sugerir Melhorias

Abra uma [Issue](https://github.com/fabriciopsouza/metacognition-framework/issues) com tag `enhancement`:

- Descreva a melhoria proposta
- Por que seria útil
- Exemplos de uso (se aplicável)

### 3. Adicionar Casos de Uso

**Muito valorizado!** Casos reais ajudam a comunidade.

**Formato esperado:**
- Usar template de [`examples/01_financial_analysis.md`](../examples/01_financial_analysis.md)
- Incluir transcrição antes/depois
- Descaracterizar dados sensíveis
- Adicionar lições aprendidas

**Como submeter:**
1. Fork o repositório
2. Criar arquivo `examples/0X_seu_caso.md`
3. Abrir Pull Request

### 4. Traduzir para Outros Idiomas

Traduções aumentam alcance global.

**Idiomas prioritários:**
- Inglês (EN)
- Espanhol (ES)
- Alemão (DE)

**Arquivos para traduzir:**
- `README.md` → `README_EN.md`
- `FRAMEWORK.md` → `FRAMEWORK_EN.md`
- `docs/GETTING_STARTED.md` → `docs/GETTING_STARTED_EN.md`

**Como submeter:**
1. Fork o repositório
2. Criar arquivos traduzidos com sufixo `_EN`, `_ES`, etc
3. Abrir Pull Request

### 5. Melhorar Documentação

Encontrou algo confuso? Erro de digitação? Link quebrado?

Abra Pull Request com a correção.

## Diretrizes

### Código de Conduta

- Seja respeitoso e profissional
- Críticas construtivas são bem-vindas
- Sem discriminação de qualquer tipo
- Discussões técnicas focadas no mérito

### Estilo de Escrita

Para manter consistência:

- **Tom:** Profissional, direto, técnico (não acadêmico demais)
- **Exemplos:** Sempre concretos, nunca genéricos
- **Validações:** Sempre que apresentar dados, rastrear fonte
- **Formatação:** Markdown limpo, sem HTML inline

### Commits

Use [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: adiciona caso de uso de machine learning
fix: corrige typo no README
docs: atualiza guia de instalação
refactor: simplifica exemplo de ETL
```

### Pull Requests

**Antes de abrir PR:**
1. Teste suas mudanças
2. Verifique ortografia/gramática
3. Confirme que links funcionam
4. Descaracterize dados sensíveis

**Ao abrir PR:**
- Título claro do que foi feito
- Descrição explicando motivação
- Se resolve issue, mencionar: "Closes #42"

---

## O Que NÃO Aceitar

❌ **Dados sensíveis/privados**
- Nomes de empresas (exceto descaracterizadas)
- Dados financeiros reais identificáveis
- Informações confidenciais

❌ **Conteúdo não relacionado**
- Marketing/spam
- Conteúdo promocional
- Links para produtos/serviços

❌ **Mudanças substanciais sem discussão**
- Alterações grandes no framework core
- Remoção de funcionalidades existentes
- Mudanças que quebram compatibilidade

Para mudanças grandes, **abra Issue primeiro** para discutir antes de implementar.

---

## Processo de Revisão

1. **Você abre PR** → aguarda revisão
2. **Mantenedor revisa** → pode pedir ajustes
3. **Você ajusta** (se necessário)
4. **PR aprovado** → merge para main
5. **Seu nome** adicionado em CONTRIBUTORS.md

**Tempo de revisão:** geralmente 2-7 dias (projeto mantido por 1 pessoa, paciência 🙏)
