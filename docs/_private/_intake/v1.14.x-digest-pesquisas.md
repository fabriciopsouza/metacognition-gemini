## ONDA 0 — P3 contrato mínimo de skill
**Obrigatórios (≤8):** `name` (^[a-z0-9-]+$, ≤64), `description` (≤1024, não-vazia), `role_order` (int), `consumes` (lista artefato+junção), `produces` (lista artefato-gate), `pass_criteria` (binário), `confidence_required` (bool), `shared_refs` (lista paths _shared/).
**Opcionais:** `version` (SemVer), `allowed-tools`, `rewind_target`, **`enforcement: {ide, chat}`** (P1 §4 — sinergia), **`classe`** salva-vidas/operacional/andaime (P7 — sinergia).
**Verificação:** IDE = JSON Schema + CI (precedente `quick_validate.py`); chat = auto-check 1 linha.
**Anti-padrão:** campo sem gate que o consuma = andaime morto (JARVIS 5/8 vazias); campo igual em todas as skills → pertence a _shared/. Limite de cerimônia: >8 obrigatórios = engordou.
**Trava P1:** campo obrigatório sem dado induz alucinação → opcional/anulável.

## ONDA 1 — P4 efeito + P1 gate
**P4 predicados E1–E6:** E1 destrói/perde dados; E2 irreversível/não-idempotente; E3 externamente visível/sai do trust boundary; E4 custa dinheiro/obrigação legal; E5 altera controle de segurança/permissão; E6 atípico/fora de escopo.
**Tiers:** T1 reversível+baixo = autonomia; T2 reversível+alto OU irreversível+baixo = ask/log; T3 irreversível+alto (E1–E2 ∩ E3–E5) = gate humano obrigatório, deny default, never auto-aprovável, four-eyes fora do canal.
**Registro auditável por gate:** ação + predicado(s) E + tier + args + identidade do agente + decisão + aprovador humano + timestamp + (regulado) before/after.
**P1 hooks Claude Code:** `PreToolUse` etc.; exit 2 = bloqueia (stderr→Claude); exit 0 = parseia JSON; **qualquer outro código (incl. exit 1) = não-bloqueante, ação PROSSEGUE**. Forma expressiva: exit 0 + `hookSpecificOutput.permissionDecision: allow|deny|ask`; precedência `deny>ask>allow`.
**Fail-closed (defense-in-depth, bug #44642):** managed-settings `disableBypassPermissionsMode:"disable"` + `allowManagedHooksOnly` + `allowManagedPermissionRulesOnly` + deny `Bash(claude *)` + teste adversarial canary em CI.
**Honestidade chat:** sem verificador externo, self-check é higiene, NUNCA "gate". `enforcement` nunca afirma paridade.
**Lit:** Saltzer&Schroeder 1975; NIST SP800-207; OWASP C3 + Agentic Top10; EU AI Act Art.14; Moldovan&Abbeel 2012.

## ONDA 3 — P5+P7 coletor único
**P5 (blame, 2 métricas só):** (a) junção-origem do rewind; (b) rounds de qa-critic até PASS. Reprovado: dashboards, latência fina, %verde, múltiplos coletores. Teste: métrica não-aberta em decisão real = dívida.
**P7 (tally + classe + poda):** campo S/N por regra no method-audit; classe salva-vidas/operacional/andaime. Poda só "andaime", N=5–10 sessões [INFERIDO, parâmetro]. Salva-vidas nunca poda por desuso (gera nota). Portão pequeno: suspender→observar→remover. Chesterton.
**Fronteira coletor único:** P5 = fluxo entre junções nesta execução; P7 = uso de regra ao longo de sessões. Mesmo artefato físico (history.md/method-audit), não dois sistemas. ADR-017 pai com 2 decisões.
**Lit:** Barrak 2025 (arXiv 2510.07614) blame attribution; Google SRE.

## ONDA 4 — P6 QA 1 turno + heterogeneidade
**Protocolo (1 turno no qa-critic existente):** STEELMAN (reconstrói versão mais forte, calibra severidade) → ATAQUE (hipótese=bug) → VEREDITO (PASSA/PASSA-COM-RESSALVA/FALHA).
**Alavanca causal real:** heterogeneidade de modelo gerador↔crítico (Zhang 2025 Heter-MAD), NÃO número de papéis. MAD não supera self-consistency.
**Self-Critique Paradox (Snorkel 2025):** forçar crítica em tarefa de alta confiança derruba acurácia 15–40% → silenciar crítico em rotina; disparar só em ambíguo/alto-impacto/regulado.
**NÃO FAZER:** Conclave 3 papéis (homogêneo reforça viés "When Debate Fails"; +3× tokens; regulado quer independência+rastreabilidade, não 3 personas).
**Lit:** Zhang 2025 (2502.08788); Huang 2023 (2310.01798); Kamoi 2024 TACL; CriticGPT (McAleese).
